## Key Design Decisions

### Data Models
- Used nullable reference types for optional fields (Url, EventDate)
- Added helper properties (IsReserved in GiftItem) for business logic
- Included audit fields (CreatedDate) for tracking

### DTOs with Validation
- Implemented DataAnnotations for validation (Required, StringLength, Url)
- Separate DTOs from entities to handle business validation rules
- Request-specific validation (e.g., GiftItem requires GiftListId)

### Interface Design
- All methods are async and return Task<T> for performance
- Repository interfaces separate from service interfaces (separation of concerns)
- Methods return nullable types (T?) for operations that may not find data
- Parameterized methods for flexible filtering

### Error Handling
- Specific exception types for different error scenarios
- Preparation for HTTP status code mapping (404 for NotFoundException)
